Aseprite

스마트문화 앱 콘텐츠 제작 이론 시간에 코드업 사이트의 문제들을 풀며 주어진 문제와 상황에 따라 알고리즘을 짤 수 있는 연습을 할 수 있었다. 다른 사람들의 답안도 보며 내 코드와 비교해 어떤 코드가 더 효율적이고 좋은 코드인지 알아보고 배우는 시간이었다.

문화 수행평가를 준비하며 메소드 오버로딩과 메소드 오버라이딩 이라는 개념에 대해 배웠다. 메소드 오버로딩은 매개 변수의 타입이나 개수가 다른 여러 개의 메소드가 같은 이름으로 작성되는 것을 말하고, 메소드 오버라이딩은 클래스 상속 상황에서 부모 class의 멤버를 자식 class에서 상속 받았지만 자식 class에서 해당 멤버의 내용을 수정하여 자식 class객체에서 적용하는 것을 말한다.

수업시간에 클래스에 대해서도 배웠다. 클래스 명을 정할 때 몇가지 규칙이 있는데,
1. 클래스 명은 하나 이상의 문자로 이루어져야 한다.
2. 반드시 대문자로 시작한다.
3. 영문 대소문자와 숫자, 특수문자로 클래스를 작성한다.
4. 클래스명은 예약어를 사용할 수 없으며, ‘$’ 와 ‘_’이외의 특수문자를 사용할 수 없다.

이클립스를 이용해 자바로 다이나믹 비트라는 리듬게임을 만들었다.
이 프로그램을 만들면서 자바에서 사진과 노래를 불러오는 방법을 배웠고,
setTitle, setSize. setResizeable 등의 여러 기능을 알게 되었다.

문화 수행평가를 하며 예외처리에 대해서도 배웠다.
JAVA에서는 try ~ catch 구문을 통해 예외 처리를 해줄 수 있다.
try : 예외가 발생할 가능성이 있는 명령문
catch : 예외처리
finally : 예외의 유무와 상관없이 실행하는 명령문

Project Title

비트나미를 이용해 윈도우에 웹 서버를 구축 하는 법을 배웠다. 비트나미는 “Bitnami WAMP Stack” 이라고 하는 패키지 소프트웨어 이다. WAMP는 Windows, Apache, MySQL, PHP의 머리글자를 모아놓은 단어이다.

CBT사이트에서 정보처리 산업기사 기출 문제들중 몇몇 문제를 풀고 정리했다. 처음에는 아무것도 몰라서 어려웠지만 계속 보면서 정리하다보니 문제가 외워지거나 이해가 돼서 조금은 할 만 해졌다.

응용프로그래밍 화면 구현 시간에 처음으로 부트스트랩을 사용해봤다. 부트스트랩이 무엇이고 어떻게 사용하는 알게 되었다. 수업시간에 부트스트랩을 배운 덕분에 따로 웹페이지를 만들 때 디자인을 더 보기 좋게 만들 수 있어서 도움이 되었다.

자신의 포트폴리오 사이트를 만들며 처음으로 다른 사람의 코드를 분석하고 내가 원하는 부분을 수정 해보는 등의 경험을 할 수 있었다. 나중에 취업을 하고 다른 사람들과의 협업이 쉬운게 아니겠구나 하는 생각도 들었고 코드를 분석해보며 몰랐던 기능도 알게 되었다.

MySQL을 통해 데이터베이스를 배웠다. 테이블을 만들고 php로 테이블의 목록과 그 테이블 안의 정보들을 가져와 출력해주는 코드를 짜보며 데이터베이스는 어떻게 사용하고 php가 뭔지 알게 되었다.
